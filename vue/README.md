# 控件库分支标准化
## 过渡阶段(临时)
    STEP1.t1时刻，基于使用者(产品)拉出使用者分支(A1...An)
    STEP2.t1时刻，基于控件类型(控件)拉出控件分支(B1...Bm)
    STEP3.t2时刻，将需要的控件分支集合合入使用者分支集合
    STEP4.t3时刻，将需要的控件分支集合合入使用者分支集合
        分支的时间无关性：控件代码、使用者代码以最新时刻为准
    
## 标准化阶段(永久)
    STEP1.t1时刻，只有一个控件主线
    STEP2.t2时刻，将需要的控件主线直接发布版本
